# 作業進捗をIssueに保存

現在の作業進捗をGitHub Issueに保存してください。

## 手順

1. **現在のtodoリストを確認**
   - 完了タスク、進行中タスク、残タスクを整理

2. **作業中のIssue番号を特定**
   - 直近のコミットログやコンテキストから特定
   - 不明な場合は `gh issue list --state open` で確認

3. **Issueに進捗コメントを追加**
   ```bash
   gh issue comment <Issue番号> --body "進捗コメント"
   ```

4. **必要に応じて中間コミットを作成**
   - 未コミットの変更がある場合は `git status` で確認
   - WIP（Work In Progress）コミットを検討

## コメントフォーマット

```markdown
## 作業進捗（中間報告）

### 完了
- [x] タスク1
- [x] タスク2

### 進行中
- [ ] タスク3（詳細状況）

### 残作業
- [ ] タスク4
- [ ] タスク5

### 次のセッションへの引き継ぎ事項
- 注意点やコンテキスト情報

---
⏸️ コンテキスト節約のため一時保存
```

## 注意事項

- 作業が完了している場合は、このコマンドではなく通常のクローズ処理を行う
- 中間保存後は新しいセッションで `/task` から再開できる
