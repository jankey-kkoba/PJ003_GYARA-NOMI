# GitHub ISSUE作業ワークフロー

## 目的
途中で作業が終了しても後で再開しやすくするため

## フロー
1. **作業予定を先にユーザーに提示する** - ISSUEの内容を確認し、実行計画を立てて提示
2. **ユーザーのOKを待つ** - 確認なしに作業を開始しない
3. **OKをもらったらISSUEにTODO記載** - .github/ISSUE_TEMPLATE/feature_request.mdのTODOセクションに実行計画を記載
4. **作業開始** - TODO記載後に実装を進める

## 注意事項
- 作業が途中で中断されても、ISSUEのTODOを見れば再開できる状態にする
- TODOは具体的なステップで記載する
